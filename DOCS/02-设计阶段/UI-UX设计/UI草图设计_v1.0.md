## 1. 设计概述

### 1.1 设计目标

基于好饭碗门店生命周期管理系统的业务需求，创建符合企业级中后台系统标准的用户界面设计，确保：

- **功能完整性**：覆盖从开店计划到门店运营的全生命周期管理
- **用户体验优化**：简化复杂业务流程，提升操作效率
- **视觉统一性**：建立一致的设计语言和视觉风格
- **设备适配性**：支持PC端、移动端和企业微信环境

### 1.2 设计原则

#### 业务导向原则
- **流程清晰**：业务流程可视化，操作路径明确
- **数据优先**：重要信息优先展示，层级结构合理
- **效率为先**：减少操作步骤，提升工作效率
- **权限明确**：不同角色看到相应的功能和数据

#### 用户体验原则
- **认知负荷**：信息密度适中，避免认知过载
- **反馈及时**：操作结果及时反馈，状态变化清晰
- **容错设计**：预防用户错误，提供友好的错误处理
- **学习成本**：界面直观易懂，操作模式一致

### 1.3 设计风格定位

- **风格定位**：现代简约、专业严谨
- **色彩倾向**：以蓝色为主色调，体现专业和信赖
- **视觉层次**：清晰的信息层级，重点突出
- **交互模式**：符合企业用户的操作习惯

## 2. 整体布局架构

### 2.1 主体布局结构

采用经典的企业级中后台布局模式：**左侧导航 + 顶部工具栏 + 主内容区**

```
┌─────────────────────────────────────────────────┐
│                   顶部工具栏                      │
│  Logo + 面包屑导航 + 用户信息 + 消息通知 + 设置    │
├──────────┬──────────────────────────────────────┤
│          │                                      │
│   侧边   │                                      │
│   导航   │            主内容区域                 │
│   菜单   │                                      │
│          │                                      │
│          │                                      │
└──────────┴──────────────────────────────────────┘
```

### 2.2 布局尺寸规范

#### PC端布局参数
```css
总体布局:
- 最小宽度: 1200px
- 推荐宽度: 1440px
- 顶部工具栏高度: 64px
- 侧边导航宽度: 256px (展开) / 80px (收起)

内容区域:
- 主内容区padding: 24px
- 页面标题高度: 56px
- 卡片间距: 16px
- 表格行高: 48px
```

#### 响应式断点
```css
断点设置:
- xs: <576px (手机竖屏)
- sm: ≥576px (手机横屏)
- md: ≥768px (平板竖屏)
- lg: ≥992px (平板横屏/小屏电脑)
- xl: ≥1200px (桌面显示器)
- xxl: ≥1600px (大屏显示器)
```

## 5. 用户流程图

### 5.1 开店全流程用户旅程

```
门店开发主流程:
  
开始 → 制定开店计划 → 录入候选点位 → 创建跟进单 → 商务谈判
  ↓
盈利测算 → 发起报店审批 → 审批通过 → 签约成功 → 创建工程单
  ↓
施工管理 → 里程碑检查 → 工程验收 → 门店交付 → 创建档案 → 开业运营

用户触点分析:
• 计划阶段: 运营人员主要使用计划管理功能
• 拓店阶段: 商务人员主要使用点位和跟进管理
• 筹备阶段: 工程和项目人员使用工程管理功能  
• 审批节点: 各层级领导使用审批中心
• 档案阶段: 多角色共同维护和查看门店信息
```

### 5.2 关键业务流程设计

#### 5.2.1 跟进管理流程
```
创建跟进单:
1. 选择候选点位 → 2. 填写基本信息 → 3. 保存跟进单

添加跟进记录:
1. 进入跟进详情 → 2. 添加跟进内容 → 3. 上传附件 → 4. 保存记录

商务条件录入:
1. 谈判结果录入 → 2. 条件对比分析 → 3. 自动盈利测算 → 4. 生成报告

发起审批:
1. 确认跟进结果 → 2. 选择审批模板 → 3. 填写申请理由 → 4. 提交审批
```

#### 5.2.2 工程管理流程
```
工程创建:
1. 基于签约信息创建 → 2. 上传设计图纸 → 3. 制定施工计划 → 4. 开始施工

进度跟踪:
1. 里程碑节点更新 → 2. 进度百分比更新 → 3. 问题记录 → 4. 甘特图显示

工程验收:
1. 选择验收类别 → 2. 核对验收项目 → 3. 标记问题项 → 4. 生成整改清单
```

#### 5.2.3 审批处理流程
```
审批处理:
1. 接收审批通知 → 2. 查看申请详情 → 3. 分析业务信息 → 4. 填写审批意见

审批操作:
• 同意: 流转到下一节点
• 驳回: 退回申请人修改  
• 转交: 转给其他人处理
• 加签: 增加审批节点
```
